# Teleport v3.5.1 内测版

> **特别注意**：因录像格式变更，此版本与旧版本的录像无法兼容播放，请勿在生产环境中升级。

## 主要更新内容：

### 核心服务
 - 升级用到的第三方库到最新版本；
 - 重构RDP协议实现，增加对各版本Win主机的兼容性；
 - 改进远程桌面远程的稳定性，减少核心服务崩溃的情况；
 - RDP远程时不会将复制的文件内容也计入录像文件中，防止录像体积暴增；
 - 完全重写RDP录像播放器，可以任意拖动RDP播放进度条了；
 - RDP播放器支持以`domain\user`格式的远程账号的录像了；
 - 改进SSH远程的稳定性，避免出现卡在登录流程上；

### WEB服务
 - 升级到Python 3.7.5；
 - 升级用到的第三方Python模块到最新版本；

### 助手
 - 升级用到的第三方库到最新版本；
 - 助手调用本地客户端，增加 `{host_name}`?变量，用于在客户端上显示远程主机名称；

## 已知问题：
 - RDP尚不支持剪贴板、磁盘映射和NLA；
 - 当TP服务器配置为 `https`?方式访问时，RDP录像播放器能否正常工作，尚未测试；
 - 未测试telnet；

## 请协助测试

### 主要测试内容

 - 能正常远程目标主机或设备；
 - 正常网络环境下，远程操作不卡顿；
 - RDP远程结束后，查看录像文件大小，正常连续操作下应为约 2MB/分钟；
 - 可以正常播放录像，可以拖动播放进度条；
 - 正在远程时，使用强行中断功能关闭远程连接，仍然可以正常播放录像；
 - 使用过程中无核心服务崩溃的现象；

### 已经测试了的环境
 - 本地Win10-64位机器
 - 远程 Win7
 - 远程 Win2008R2
 - 远程 CentOS7
 - 远程 Ubuntu16.06

### 需要的远程主机
 - Windows2000 Server 及以上版本；
 - 支持 SSHv2 的任意Linux主机或网络设备；
 - 使用 telnet 的网络设备；

### 需要的本地工作机
因为新的RDP播放器使用Qt开发，需要验证助手安装包中打包了足够的依赖的动态库。
 - Win7及以上版本
